# Triumvirate LLM Bot v2.2

Автономный бот для игры в трёхсторонние шахматы на платформе [triumvirate4llm.com](https://triumvirate4llm.com). Бот подключается к шахматной арене, получает состояние доски и использует большие языковые модели (LLM) для выбора ходов в реальном времени.

## Что это

Графический клиент-бот, который выступает мостом между шахматным сервером Triumvirate и любым LLM-провайдером. На каждом ходу бот:

1. Получает текущее состояние доски от сервера (позиция, легальные ходы, информация о шахе)
2. Формирует многослойный промпт с описанием позиции
3. Отправляет промпт в выбранную языковую модель
4. Парсит ответ модели и извлекает ход
5. Отправляет ход на сервер

Весь процесс полностью автоматизирован — после нажатия кнопки «Запустить» бот играет самостоятельно до конца партии.

## Шахматная арена

Игра ведётся на 96-клеточной гексагональной доске (колонки A–L, строки 1–12). Участвуют три игрока: White, Black, Red. Ходы делаются по очереди: White → Black → Red. Бот получает один из трёх цветов при подключении к игре.

## Поддерживаемые LLM-провайдеры

- **Ollama** — локальные модели, `http://localhost:11434/v1`
- **OpenAI API** — GPT-4o, GPT-4o-mini и другие модели OpenAI
- **Anthropic** — Claude (нативный протокол, не OpenAI-совместимый)
- **OpenRouter** — агрегатор 200+ моделей с единым API, автоматическое отслеживание стоимости
- **LM Studio** — локальные модели через LM Studio, `http://localhost:1234/v1`
- **Кастомный URL** — любой OpenAI-совместимый эндпоинт

При переключении провайдера автоматически подставляются базовый URL, модель по умолчанию и специфичные заголовки. API-ключи могут загружаться из переменных окружения: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `OPENROUTER_API_KEY`.

## Форматы ответа LLM

- **Простой текст** — модель отвечает строкой вида `E2 E4`
- **JSON** — модель отвечает объектом `{"from": "E2", "to": "E4"}`
- **JSON с рассуждением** — модель сначала анализирует позицию: `{"thinking": "анализ...", "from": "E2", "to": "E4"}`

Формат с рассуждением даёт моделям возможность «думать» перед ходом, что улучшает качество игры.

## Система надёжности

- **Ретраи с эскалацией** — при нелегальном ходе бот повторяет запрос к LLM (до N попыток). На промежуточных попытках добавляет контекст ошибки. На последней попытке радикально упрощает запрос, давая прямой список легальных ходов
- **Фоллбэк на случайный ход** — если все попытки LLM исчерпаны, бот выбирает случайный легальный ход (можно отключить)
- **Двойной парсинг** — ответ модели парсится сначала как JSON, затем как текст через регулярные выражения. Обрабатываются префиксы фигур, промоушен пешек, нестандартное форматирование

## Отслеживание стоимости

Для OpenRouter автоматически загружаются тарифы модели. На каждый ход считается стоимость по input/output/reasoning токенам. Накопительная статистика выводится в лог при остановке бота.

## Трассировка ходов

Каждый ход сохраняется как отдельный JSON-файл в директории `logs/`. Содержит полную цепочку:

- Сырое состояние доски от сервера
- Отрендеренный промпт (системный + пользовательский)
- Все запросы и ответы LLM с количеством токенов и стоимостью
- Попытки парсинга (какие координаты найдены, какие пары проверены)
- Выбранный ход
- Ответ сервера на ход

Файлы организованы по играм: `logs/game_<id>__<model>/move_001.json`.

## Настраиваемые промпты

- **Системный промпт** — описание роли и правил для модели
- **Шаблон пользователя** — шаблон с плейсхолдерами `{{move_number}}`, `{{current_player}}`, `{{legal_moves}}`, `{{board}}`, `{{check}}` и другими
- **Дополнительные правила** — произвольные инструкции, добавляемые к системному промпту

Промпты редактируются прямо в интерфейсе. Также могут загружаться из файлов `prompts/system_prompt.txt` и `prompts/user_prompt_template.txt`.

## Графический интерфейс

Построен на NiceGUI. Левая панель — настройки (провайдер, модель, промпты, параметры бота). Правая панель — три вкладки:

- **Игра** — текущее состояние доски, легальные ходы, последние события
- **Лог** — полный лог с копированием в буфер и очисткой
- **Лобби** — список активных игр на сервере

Кнопки диагностики: «Ping сервер» и «Проверить LLM» позволяют убедиться в работоспособности до запуска игры.

## Именование бота

Имя бота автоматически формируется из провайдера и модели, например `LLM_OpenRouter_openai/gpt-4.1-nano`. Удобно при параллельном запуске нескольких экземпляров. Авто-именование можно отключить и задать имя вручную.

## Персистентные настройки

Все параметры сохраняются в `llm_bot_gui_settings_v2.json` и восстанавливаются при следующем запуске. Флаг `--settings` позволяет указать отдельный файл для каждого экземпляра бота.

## Установка

```bash
pip install nicegui httpx websockets
```

## Запуск

```bash
# Десктопное окно (1440×900)
python main.py

# Веб-сервер на порту 8090
python main.py --web

# Веб-сервер на другом порту
python main.py --web --port 9000

# Несколько ботов одновременно
python main.py --web --port 8090 --settings bot_gpt4.json
python main.py --web --port 8091 --settings bot_claude.json
```

Для headless-серверов (Ubuntu без GUI):

```bash
python main.py --web --host 0.0.0.0 --port 8090
```

Затем открыть `http://<ip-сервера>:8090` в браузере.

## Структура проекта

```
├── main.py              — точка входа, argparse
├── constants.py         — провайдеры, ключи окружения, версия
├── settings.py          — настройки (JSON), дефолтные промпты
├── llm_client.py        — HTTP-клиент для LLM (OpenAI + Anthropic)
├── arena_client.py      — HTTP-клиент REST API арены
├── pricing.py           — тарифы OpenRouter, расчёт стоимости
├── prompt_builder.py    — сборка промптов из шаблонов
├── move_parser.py       — парсинг ответа LLM → шахматный ход
├── tracer.py            — трассировка ходов → JSON-логи
├── bot_runner.py        — основной игровой цикл (asyncio)
├── gui.py               — NiceGUI интерфейс
├── CLAUDE.md            — описание структуры для Claude Code
└── prompts/
    ├── system_prompt.txt
    └── user_prompt_template.txt
```
