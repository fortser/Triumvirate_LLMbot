## 1. ВНЕШНИЙ КОНТУР ДОСКИ

### 1.1. Тип фигуры

Доска представляет собой **правильный шестиугольник** в ориентации **flat-top** (плоская сторона сверху и снизу, вершины слева и справа).

Правильный шестиугольник означает: все 6 сторон равны, все 6 внутренних углов равны 120°.
 

### 1.3. Шесть вершин шестиугольника

Вершины вычисляются от центра по формуле:

```
Vn = (CX + HEX_R × cos(angle), CY − HEX_R × sin(angle))
```

Знак минус перед sin обусловлен тем, что в экранных координатах ось Y направлена вниз.

| Вершина | Угол (градусы) | Примерные координаты (px) | Положение на экране |
|---------|---------------|--------------------------|---------------------|
| **V1** | 210° | (162, 685) | Нижняя левая |
| **V2** | 150° | (162, 295) | Верхняя левая |
| **V3** | 90°  | (500, 100) | Верхняя центральная |
| **V4** | 30°  | (838, 295) | Верхняя правая |
| **V5** | 330° | (838, 685) | Нижняя правая |
| **V6** | 270° | (500, 880) | Нижняя центральная |

### 1.4. Шесть сторон шестиугольника

| Сторона | Вершины | Ориентация |
|---------|---------|------------|
| Верхняя левая | V2 → V3 | Наклонная, идёт вверх-вправо |
| Верхняя правая | V3 → V4 | Наклонная, идёт вниз-вправо |
| Правая | V4 → V5 | Почти вертикальная, идёт вниз |
| Нижняя правая | V5 → V6 | Наклонная, идёт вниз-влево |
| Нижняя левая | V6 → V1 | Наклонная, идёт вверх-влево |
| Левая | V1 → V2 | Почти вертикальная, идёт вверх |

Все сопряжения сторон — **угловые, без скругления** (R = 0).


## 2. ДЕЛЕНИЕ НА ТРИ СЕКТОРА

### 2.1. Принцип деления

Шестиугольник делится на **три равных сектора**. Каждый сектор имеет форму **трапеции** (точнее — равнобедренной трапеции, деформированной геометрией шестиугольника).

Деление происходит **тремя линиями**, идущими от центра доски к трём точкам на периметре. Эти три точки — **середины трёх чередующихся сторон** шестиугольника.

### 2.2. Три разделительные линии

| Разделитель | Начало | Конец | Разделяет секторы |
|-------------|--------|-------|-------------------|
| Линия 1 | Центр (500, 490) | Середина стороны V2–V3 = mid(V2, V3) | White и Black |
| Линия 2 | Центр (500, 490) | Середина стороны V4–V5 = mid(V4, V5) | Black и Red |
| Линия 3 | Центр (500, 490) | Середина стороны V6–V1 = mid(V6, V1) | Red и White |

Координаты конечных точек разделителей:

| Точка | Вычисление | Примерные координаты |
|-------|-----------|---------------------|
| mid(V2, V3) | ((162+500)/2, (295+100)/2) | (331, 198) |
| mid(V4, V5) | ((838+838)/2, (295+685)/2) | (838, 490) |
| mid(V6, V1) | ((500+162)/2, (880+685)/2) | (331, 783) |

Три разделителя расходятся от центра под углами **120°** друг к другу — это обеспечивает равенство секторов.

- **base_mid** = mid(BL, BR) — середина базы

## 3. СЕТКА КЛЕТОК: АЛГОРИТМ ПОСТРОЕНИЯ

### 3.1. Параметры сетки

| Параметр | Значение |
|----------|----------|
| Столбцов в секторе | **8** (нумерация col = 0..7) |
| Рядов в секторе | **4** (нумерация row = 0..3) |
| Клеток в секторе | 8 × 4 = 32 (включая 2 розеточных) |
| Обычных клеток в секторе | **30** |
| Розеточных клеток в секторе | **2** (col=3 и col=4 при row=3) |
| Всего клеток на доске | 30 × 3 + 6 = **96** |

### 3.2. Ориентация осей внутри сектора

```
    col=0  col=1  col=2  col=3  col=4  col=5  col=6  col=7
     │      │      │      │      │      │      │      │
     ▼      ▼      ▼      ▼      ▼      ▼      ▼      ▼

    ┌──────┬──────┬──────┬──────┬──────┬──────┬──────┬──────┐ ← row=0 (тыл, база)
    │      │      │      │      │      │      │      │      │
    ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤ ← row=1
    │      │      │      │      │      │      │      │      │
    ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤ ← row=2
    │      │      │      │      │      │      │      │      │
    ├──────┼──────┼──────┼──────┼──────┼──────┼──────┼──────┤ ← row=3 (фронт)
    │      │      │  ◈◈  │  ◈◈  │      │      │      │      │
    └──────┴──────┴──────┴──────┴──────┴──────┴──────┴──────┘
                          ↓  ↓
                        ЦЕНТР ДОСКИ (розетка)
```

Важные определения:

- **row = 0** — это **тыл** (внешний край доски, база трапеции, самые широкие клетки)
- **row = 3** — это **фронт** (ближе к центру, самые узкие клетки)
- **col = 0** — левый край сектора (ближе к левому сопернику)
- **col = 7** — правый край сектора (ближе к правому сопернику)

### 3.3. Основной алгоритм: функция sector_grid_point

Для вычисления координат любого узла сетки (col, row) используется **трёхточечная линейная интерполяция с изломом**:

```
ВХОД: col ∈ [0..8], row ∈ [0..4]  (узлы сетки, на 1 больше чем клеток)

ШАГ 1: Вычислить параметр глубины
    t = row / 4    (0.0 = база, 1.0 = центр доски)

ШАГ 2: Три опорные точки для текущего ряда
    left_pt  = lerp(BL, side_left, t)     — левый край ряда
    apex_pt  = lerp(base_mid, center, t)  — центральная точка ряда (излом)
    right_pt = lerp(BR, side_right, t)    — правый край ряда

ШАГ 3: Вычислить параметр столбца
    u = col / 8    (0.0 = левый край, 1.0 = правый край)

ШАГ 4: Интерполяция с изломом в центре
    ЕСЛИ u ≤ 0.5:
        point = lerp(left_pt, apex_pt, u × 2)
    ИНАЧЕ:
        point = lerp(apex_pt, right_pt, (u − 0.5) × 2)
```

Где lerp — линейная интерполяция:

```
lerp(P1, P2, t) = (P1.x + (P2.x − P1.x) × t,  P1.y + (P2.y − P1.y) × t)
```

### 3.4. Почему «с изломом»

Каждый ряд клеток представляет собой **не один прямой отрезок**, а **два отрезка с изломом в середине**. Это связано с тем, что сектор-трапеция имеет вершину (центр доски) по одну сторону и два угла базы — по другую. Линия, проходящая от left_pt к right_pt через apex_pt, образует «угол» (излом) в середине. Этот излом:

- **Максимален при row = 0** (база) — apex_pt = base_mid = середина отрезка BL–BR, излома фактически нет (три точки коллинеарны)
- **Нарастает с увеличением row** — по мере приближения к центру, left_pt и right_pt расходятся к боковым сторонам, а apex_pt стремится к центру доски
- **Максимален при row = 4** — apex_pt = center, left_pt = side_left, right_pt = side_right

Именно этот излом создаёт **ромбовидную (изометрическую) форму клеток** вместо прямоугольной.

### 3.5. Четыре вершины каждой клетки

Клетка (col, row) определяется четырьмя узлами сетки:

```
    p00 = sector_grid_point(col,     row)       — верхний левый
    p10 = sector_grid_point(col + 1, row)       — верхний правый
    p11 = sector_grid_point(col + 1, row + 1)   — нижний правый
    p01 = sector_grid_point(col,     row + 1)   — нижний левый
```

Четырёхугольник (p00, p10, p11, p01) — это **параллелограмм / трапецоид**, форма которого меняется в зависимости от позиции.

### 3.6. Деформация клеток

```
                            ШИРИНА                    ФОРМА
    row=0 (тыл/база):      Максимальная              Почти прямоугольники
    row=1:                  ~75% от базовой           Лёгкие трапеции
    row=2:                  ~50% от базовой           Заметные трапеции
    row=3 (фронт):         ~25% от базовой           Узкие трапеции/ромбы
    (розетка):             Минимальная                Клиновидные фигуры
```

Количественно: ширина ряда пропорциональна (1 − t), где t = row/4. При row=3 ширина составляет 25% от базовой, при row=4 (гипотетическая точка розетки) — 0% (всё сходится в центр).

## 4. РОЗЕТКА

### 4.1. Какие клетки образуют розетку

Розетка — это **6 клеток**, которые занимают позиции **row=3, col=3 и col=4** в каждом из трёх секторов. Эти клетки находятся на самом фронте каждого сектора, в двух центральных столбцах.

| Клетка розетки | Сектор | col | row | Между какими секторами |
|----------------|--------|-----|-----|------------------------|
| Розетка 1 (W, col=3) | White | 3 | 3 | White–Black |
| Розетка 2 (W, col=4) | White | 4 | 3 | White–Red |
| Розетка 3 (B, col=3) | Black | 3 | 3 | Black–Red |
| Розетка 4 (B, col=4) | Black | 4 | 3 | Black–White |
| Розетка 5 (R, col=3) | Red | 3 | 3 | Red–White |
| Розетка 6 (R, col=4) | Red | 4 | 3 | Red–Black |

### 4.2. Геометрия розеточных клеток

Розеточные клетки вычисляются **тем же алгоритмом** sector_grid_point, что и обычные. Но благодаря тому, что при row=3→4 сетка сходится к центру, эти клетки:

- Имеют **клиновидную** форму (узкая сторона ближе к центру)
- Существенно **меньше** обычных клеток
- Все 6 штук **плотно окружают** центральную точку доски
- Образуют характерный **шестилепестковый узор**


Раздел 7.2. Шахматная раскраска клеток
7.2.1. Назначение

Шахматная раскраска обеспечивает визуальное различение смежных клеток путём чередования двух цветов (светлого и тёмного). Любые две клетки, имеющие общую сторону, должны иметь противоположные цвета.
7.2.2. Проблема согласования между секторами

Доска TRIUMVIRATE состоит из трёх секторов, каждый из которых имеет собственную локальную систему координат (col, row). Секторы сходятся в центре доски, где расположена розетка — шесть клеток, по две от каждого сектора.

Проблема: Наивное применение формулы (col + row) % 2 независимо в каждом секторе приводит к нарушению чередования на границах секторов. Смежные розеточные клетки из разных секторов могут получить одинаковый цвет.

Причина: Три сектора ориентированы под углами 120° друг к другу. При нечётном количестве секторов один из них оказывается в «инвертированной» позиции относительно двух других.

Решение: Ввести корректирующее смещение паритета для сектора White.
7.2.3. Смещение паритета по секторам
Сектор 	Смещение паритета (parity_offset)
White 	1
Black 	0
Red 	0
7.2.4. Формула вычисления паритета

parity = (col + row + parity_offset) % 2

Параметры:

    col — номер столбца клетки, диапазон 0–7
    row — номер ряда клетки, диапазон 0–3
    parity_offset — смещение паритета для сектора (см. таблицу 7.2.3)

Результат:

    parity = 0 → светлый цвет
    parity = 1 → тёмный цвет

7.2.5. Примеры вычисления
Пример 1: Обычная клетка сектора Black

Вход: сектор = Black, col = 2, row = 1

Вычисление:

parity_offset = 0 (для Black)
parity = (2 + 1 + 0) % 2 = 3 % 2 = 1

Результат: тёмный цвет

Самопроверка: Соседняя клетка (col=3, row=1) того же сектора:

parity = (3 + 1 + 0) % 2 = 4 % 2 = 0 → светлый ✓ (отличается)

Пример 2: Обычная клетка сектора White

Вход: сектор = White, col = 2, row = 1

Вычисление:

parity_offset = 1 (для White)
parity = (2 + 1 + 1) % 2 = 4 % 2 = 0

Результат: светлый цвет

Самопроверка: Соседняя клетка (col=2, row=2) того же сектора:

parity = (2 + 2 + 1) % 2 = 5 % 2 = 1 → тёмный ✓ (отличается)

7.2.6. Верификация на розеточных клетках

Розетка состоит из шести клеток (row=3, col∈{3,4} для каждого сектора). Эти клетки физически окружают центр доски и должны образовывать непрерывный чередующийся паттерн.
Таблица паритетов розеточных клеток
Сектор 	col 	row 	parity_offset 	Вычисление 	parity 	Цвет
White 	3 	3 	1 	(3+3+1) % 2 = 1 	1 	тёмный
White 	4 	3 	1 	(4+3+1) % 2 = 0 	0 	светлый
Black 	3 	3 	0 	(3+3+0) % 2 = 0 	0 	светлый
Black 	4 	3 	0 	(4+3+0) % 2 = 1 	1 	тёмный
Red 	3 	3 	0 	(3+3+0) % 2 = 0 	0 	светлый
Red 	4 	3 	0 	(4+3+0) % 2 = 1 	1 	тёмный
Физическое расположение розеточных клеток

Секторы расположены на доске следующим образом:

    Black — верхняя часть доски
    White — нижняя левая часть доски
    Red — нижняя правая часть доски

Розеточные клетки образуют кольцо вокруг центра. Порядок обхода по часовой стрелке, начиная с верхней левой позиции:

4 	R(4,3) 	тёмный 	→ 	W(4,3) 	светлый 	✓
5 	W(4,3) 	светлый 	→ 	W(3,3) 	тёмный 	✓
6 	W(3,3) 	тёмный 	→ 	B(3,3) 	светлый 	✓

Результат: Все смежные пары имеют противоположные цвета. Паттерн непрерывен.
7.2.7. Цветовая палитра
Сектор 	Светлый (parity=0) 	Тёмный (parity=1) 	Контур
White 	#dcdcc8 	#c0c0a8 	#8a8a6e
Black 	#c8c8de 	#a8a8c4 	#6e6e8e
Red 	#dec8c8 	#c4a8a8 	#8e6e6e
Розетка 	#fff4c8 	#ffe090 	#c89830

Примечание: Розеточные клетки (row=3, col∈{3,4}) используют цвета розетки вместо цветов своего сектора.
7.2.8. Алгоритм определения цвета клетки

ВХОД:
    sector — идентификатор сектора ("White", "Black", "Red")
    col — номер столбца (0–7)
    row — номер ряда (0–3)

КОНСТАНТЫ:
    PARITY_OFFSET = {
        "White": 1,
        "Black": 0,
        "Red": 0
    }
    
    SECTOR_COLORS = {
        "White": {0: "#dcdcc8", 1: "#c0c0a8"},
        "Black": {0: "#c8c8de", 1: "#a8a8c4"},
        "Red":   {0: "#dec8c8", 1: "#c4a8a8"}
    }
    
    ROSETTE_COLORS = {0: "#fff4c8", 1: "#ffe090"}

АЛГОРИТМ:
    1. Получить смещение паритета для сектора:
       offset = PARITY_OFFSET[sector]
    
    2. Вычислить паритет клетки:
       parity = (col + row + offset) % 2
    
    3. Определить, является ли клетка розеточной:
       is_rosette = (row == 3) AND (col == 3 OR col == 4)
    
    4. Выбрать цвет:
       ЕСЛИ is_rosette:
           color = ROSETTE_COLORS[parity]
       ИНАЧЕ:
           color = SECTOR_COLORS[sector][parity]
    
    5. Вернуть color

ВЫХОД:
    color — hex-код цвета заливки клетки

7.2.9. Сводка ключевых правил

    Базовая формула: parity = (col + row + parity_offset) % 2

    Смещения паритета: White = 1, Black = 0, Red = 0

    Интерпретация паритета: 0 = светлый, 1 = тёмный

    Розеточные клетки: row = 3 при col ∈ {3, 4} — используют отдельную палитру

    Инвариант: Любые две смежные клетки (включая клетки на границе секторов) имеют противоположный паритет


### 7.6. Подсветка при наведении

При наведении курсора на клетку:

- Контур клетки обводится **жёлтым** (#ffe840) с толщиной 3 px
