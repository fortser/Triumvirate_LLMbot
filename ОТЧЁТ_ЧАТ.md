# Отчёт: Реализация чата между ботами во время игры

**Дата:** 2026-03-19
**Коммит сервера:** `e8e90b7`
**Статус:** Реализовано, протестировано, совместимость проверена

---

## Описание функционала

Добавлена возможность отправлять **одно текстовое сообщение (до 256 символов)** вместе с ходом. Сообщения видны всем участникам и зрителям. Боты, управляемые языковыми моделями, могут вести переговоры, блефовать и формировать союзы прямо во время игры.

**Ключевые решения:**
- Сообщение — необязательное поле в запросе хода (обратная совместимость)
- Одно сообщение за ход (привязано к `POST /move`)
- Лимит 256 символов (защита от prompt injection)
- В ответе `/state` — последние 6 сообщений (2 полных раунда)
- В памяти сервера — буфер на 20 сообщений
- В БД — столбец `chat_message` в таблице `moves`

## ЧАСТЬ 1: Серверная часть (Triumvirate4LLM)

### Изменённые файлы

#### 1. `server/app/config.py` — Настройки чата
```python
CHAT_MAX_LENGTH: int = 256      # Максимальная длина сообщения
CHAT_HISTORY_DEPTH: int = 6     # Глубина в ответе /state (2 раунда)
```

#### 2. `server/app/models/requests.py` — Поле `message` в MoveRequest
```python
class MoveRequest(BaseModel):
    from_square: str = Field(alias="from", ...)
    to_square: str = Field(alias="to", ...)
    move_number: int
    promotion: str | None = None
    message: str | None = Field(None, max_length=256)  # ← НОВОЕ
```
- Pydantic автоматически возвращает HTTP 422 при превышении 256 символов
- Поле необязательное — все старые клиенты работают без изменений

#### 3. `server/app/game/session.py` — ChatMessage и буфер
```python
@dataclass
class ChatMessage:
    move_number: int
    color: str
    player_name: str
    message: str
```
- `self.chat_history: deque[ChatMessage] = deque(maxlen=20)` — in-memory буфер
- `add_chat_message(move_number, color, message)` — добавление сообщения
- `get_recent_chat(depth=6)` — получение последних N сообщений как list[dict]

#### 4. `server/app/models/responses.py` — Поле `chat_history` в GameStateResponse
```python
class GameStateResponse(BaseModel):
    # ... все существующие 14 полей ...
    server_version: str
    chat_history: list[dict] = Field(default_factory=list)  # ← НОВОЕ
```
- `default_factory=list` — обратная совместимость: клиенты без chat_history получат `[]`

#### 5. `server/app/serializers/adapter.py` — Заполнение chat_history
```python
return GameStateResponse(
    # ... все существующие аргументы ...
    chat_history=session.get_recent_chat(settings.CHAT_HISTORY_DEPTH),  # ← НОВОЕ
)
```

#### 6. `server/app/api/router.py` — Три изменения в `/move`

**6a. Сохранение сообщения после успешного хода (внутри lock):**
```python
if body.message:
    session.add_chat_message(body.move_number, slot.color, body.message)
```

**6b. Broadcast payload с chat_message:**
```python
broadcast_payload = {"type": "move", "state": ..., ...}
if body.message:
    broadcast_payload["chat_message"] = body.message
await session.broadcast_to_spectators(broadcast_payload)
```

**6c. Запись в БД:**
```python
await insert_move(conn, session.game_id, {
    # ... все существующие поля ...
    "chat_message": body.message,  # ← НОВОЕ
})
```

**Что НЕ сохраняет сообщение:**
- `forced_random` — невалидный ход (бот не заслужил чата)
- `timeout` — бот не отправлял запрос
- SmartBot — вызывает `make_move()` напрямую, без API

## ЧАСТЬ 2: Клиентская часть (Triumvirate_LLMbot)

### Изменённые файлы

#### 1. `arena_client.py` — Параметр `message` в `make_move()`
```python
async def make_move(
    self, from_sq: str, to_sq: str, move_number: int,
    promotion: str | None = None, message: str | None = None,  # ← НОВОЕ
) -> tuple[int, Any]:
    body = {"from": from_sq, "to": to_sq, "move_number": move_number}
    if promotion:
        body["promotion"] = promotion
    if message:
        body["message"] = message  # ← НОВОЕ
```
- Обратная совместимость: `message=None` по умолчанию
- Старый вызов `make_move(from, to, num, promo)` работает без изменений

#### 2. `move_parser.py` — Метод `extract_message()`
```python
def extract_message(self, text: str) -> str | None:
    """Extract optional chat message from LLM JSON response.
    Looks for a "message" key in the JSON object (up to 256 chars).
    """
```
- Работает для JSON и json_thinking форматов
- Извлекает поле `"message"` из JSON-ответа LLM
- Обрезает до 256 символов
- Возвращает `None` если поле отсутствует или ответ не JSON

#### 3. `prompt_builder.py` — Chat history в промпте
**Парсинг chat_history из state:**
```python
chat_history = state.get("chat_history", [])
chat_text = ""
if chat_history:
    chat_lines = []
    for msg in chat_history:
        c = msg.get("color", "?").upper()
        name = msg.get("player_name", c)
        text = msg.get("message", "")
        chat_lines.append(f"  [{name} ({c})]: {text}")
    chat_text = "\n".join(chat_lines)
```

**Подстановка в шаблон:**
- Новый плейсхолдер `{chat}` в `subs` dict
- Автоматическая вставка блока `Chat:` если `{chat}` нет в шаблоне

#### 4. `bot_runner.py` — Извлечение и отправка сообщений
**Новый атрибут:**
```python
self._last_llm_raw: str = ""  # Сырой ответ LLM для извлечения message
```

**Сохранение сырого ответа при успешном парсинге:**
```python
if result:
    self._last_llm_raw = raw or ""
    return result
```

**Извлечение message перед отправкой хода:**
```python
chat_msg: str | None = None
if not is_fallback and self._last_llm_raw:
    chat_msg = self.parser.extract_message(self._last_llm_raw)

# ... формирование move_req ...
if chat_msg:
    move_req["message"] = chat_msg

code, data = await self.arena.make_move(
    from_sq, to_sq, move_num, promo, chat_msg
)
```

**Логирование:**
```python
if code == 200:
    if chat_msg:
        self._log(f"💬 Сообщение: «{chat_msg[:80]}»")
```

## Поток данных (End-to-End)

```
LLM ──JSON──→ MoveParser ──(from,to,promo)──→ BotRunner
                  │                                │
                  └── extract_message() ───────────┘
                                                   │
                                          (from, to, promo, message)
                                                   │
                                                   ▼
                                            ArenaClient.make_move()
                                                   │
                                        POST /move {from, to, move_number, message}
                                                   │
                                                   ▼
                                          Server router.py
                                                   │
                                    ┌──────────────┼──────────────┐
                                    │              │              │
                              session.add_    broadcast       insert_move
                              chat_message    {chat_message}  (chat_message)
                                    │              │              │
                                    ▼              ▼              ▼
                                 deque(20)     WebSocket       PostgreSQL
                                    │
                                    ▼
                              GET /state → chat_history: [{...}, ...]
                                    │
                                    ▼
                              PromptBuilder.build() → промпт с Chat:
                                    │
                                    ▼
                                   LLM
```
