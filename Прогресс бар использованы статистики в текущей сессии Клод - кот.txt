# 🔍 Ситуация с прогресс-барами в Claude Code (март 2026) и все способы отслеживания лимитов

## **Что произошло с прогресс-барами:**

2 марта 2026 года пользователи заметили, что из вкладки **Settings → Usage** исчезли прогресс-бары сессии и недельных лимитов. Судя по Reddit-треду и комментарию автоматического модератора сообщества, это был **баг, а не намеренное решение** — Anthropic в тот момент переживала серьёзную нестабильность серверов. В итоге бары вернулись, но нестабильно: у части пользователей они периодически пропадают снова.

Главный вывод сообщества: **не стоит полагаться только на веб-интерфейс**. Для Claude Code CLI существуют более надёжные и информативные способы отслеживания лимитов — встроенные и сторонние.

---

## **Способ 1 — Встроенная команда `/usage` (самый простой):**

Прямо внутри любой сессии Claude Code введи:

```bash
/usage
```

Выводит три ключевых показателя:
- **Current Session Usage** — использование текущей 5-часовой сессии + таймер до сброса
- **Current Week (All Models)** — недельный лимит по всем моделям (Sonnet, Haiku) + дата сброса
- **Current Week (Opus)** — отдельный недельный лимит для Opus

Это **нулевая настройка**, работает всегда, даже когда веб-интерфейс сломан.

---

## **Способ 2 — Встроенный statusline (постоянная панель внизу терминала):**

Это именно то, что ты ищешь — **постоянная строка состояния внизу экрана Claude Code**, которая обновляется автоматически после каждого ответа. Anthropic добавила эту функцию официально.

**Самый быстрый способ включить её** — использовать встроенную команду:

```bash
/statusline show model name and context percentage with a progress bar
```

Claude Code сам сгенерирует скрипт и обновит настройки. Можно описать на русском или английском что хочешь видеть.

**Ручная настройка** — добавь в файл `~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "jq -r '[(.model.display_name), (.context_window.used_percentage // 0|tostring + \"%\")]|join(\" | \")'",
    "padding": 2
  }
}
```

Данные, доступные для отображения в statusline (приходят через stdin в формате JSON):

- `model.display_name` — текущая модель (Opus, Sonnet и т.д.)
- `context_window.used_percentage` — % использования контекстного окна
- `context_window.remaining_percentage` — % оставшегося контекста
- `context_window.context_window_size` — максимальный размер окна в токенах
- `cost.total_cost_usd` — стоимость текущей сессии в USD
- `cost.total_duration_ms` — время сессии в миллисекундах
- `cost.total_lines_added` / `total_lines_removed` — изменённые строки кода

> ⚠️ **Важно:** встроенный statusline показывает данные **текущей сессии** (контекст, стоимость), но **не показывает** процент использования 5-часового и недельного лимитов — для этого нужен ccusage.

## **Способ 3 — ccusage + statusline (лучший вариант):**

**ccusage** — популярный сторонний open-source инструмент, который читает локальные JSONL-файлы Claude Code и показывает детальную аналитику. Главная фишка — интеграция с нативным statusline Claude Code, которая даёт **постоянную панель с 5-часовым блоком и burn rate**.

**Установка:**

```bash
# Через npx (рекомендуется, ничего устанавливать не надо):
npx ccusage@latest

# Или через bun:
bun x ccusage
```

**Подключение к statusline** — добавь в `~/.claude/settings.json`:

```json
{
  "statusLine": {
    "type": "command",
    "command": "npx -y ccusage statusline",
    "padding": 0
  }
}
```

**Что будет отображаться в нижней панели:**

```
🤖 Opus | 💰 $0.23 session / $1.23 today / $0.45 block (2h 45m left) | 🔥 $0.12/hr | 🧠 25,000 (12%)
```

Расшифровка каждого элемента:
- `🤖 Opus` — текущая модель
- `$0.23 session` — стоимость текущего разговора
- `$1.23 today` — суммарная стоимость за сегодня
- `$0.45 block (2h 45m left)` — **стоимость текущего 5-часового блока и оставшееся время до сброса**
- `🔥 $0.12/hr` — скорость расхода токенов (зелёный/жёлтый/красный)
- `🧠 25,000 (12%)` — использование контекстного окна

**С визуальными индикаторами горения:**

```json
{
  "statusLine": {
    "type": "command",
    "command": "npx -y ccusage statusline --visual-burn-rate emoji",
    "padding": 0
  }
}
```

**Проверить использование прямо из терминала (без сессии):**

```bash
npx ccusage daily      # использование по дням
npx ccusage blocks     # анализ по 5-часовым блокам
npx ccusage session    # по сессиям
```

## **Способ 4 — ccstatusline (расширенный statusline с виджетами):**

Ещё один популярный проект от сообщества — **ccstatusline** на GitHub (`github.com/pcvelz/ccstatusline`). Поддерживает виджеты использования лимитов (5-часовых и недельных), скорость токенов, git-статус и многое другое. Упоминается в официальной документации Anthropic как пример community-проекта.

---

## **Итоговое сравнение:**

| Инструмент | Что показывает | Настройка |
|---|---|---|
| `/usage` (встроен) | Сессия + неделя + таймер сброса | Никакой |
| `/statusline` (встроен) | Контекст, модель, стоимость сессии | Минимальная |
| **ccusage statusline** | Стоимость, 5ч-блок с таймером, burn rate | `settings.json` |
| ccstatusline | Лимиты + скорость + git | Установка |

Для большинства сценариев оптимальная связка — это **`/usage` для быстрой проверки** и **ccusage в statusline для постоянного мониторинга** внизу экрана.
