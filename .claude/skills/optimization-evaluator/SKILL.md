# Optimization Evaluator

Скилл для оценки эффективности промпт-оптимизаций LLM-бота для Three-Player Chess.

## Когда использовать

- После прогона тестовых игр на новой версии промптов
- Для записи baseline перед оптимизацией
- Для просмотра статуса всех фаз оптимизации
- Для сравнения двух произвольных фаз

## Отличие от других агентов

| Агент | Фокус |
|-------|-------|
| `/evaluate-models` | Рейтинг моделей: кто лучше играет |
| `/analyze-strategy` | Шахматное содержание: как модели думают |
| `/optimize-prompts` | Аудит промптов: что изменить |
| **`/evaluate-optimization`** | **Эффект изменений: что изменилось после оптимизации** |

## Режимы работы

### 1. Запись baseline (по умолчанию, если лог пуст)

```
/evaluate-optimization
/evaluate-optimization baseline
```

Записывает текущие метрики 3 моделей тестового стенда как точку отсчёта.

### 2. Оценка фазы

```
/evaluate-optimization OPT-0
/evaluate-optimization OPT-1 "Chat Diplomacy"
```

Сравнивает текущие метрики с baseline и предыдущей фазой.
Вычисляет дельты, значимость, Go/No-Go.
Записывает результат в лог.

### 3. Статус всех фаз

```
/evaluate-optimization status
```

Сводная таблица всех фаз: метрики, решения, кумулятивный эффект.

### 4. Сравнение двух фаз

```
/evaluate-optimization compare OPT-1 OPT-3
```

Дельты между двумя произвольными фазами.

## Данные

| Источник | Путь | Назначение |
|----------|------|------------|
| Метрики моделей | `logs/evaluations/model_rankings.json` | Текущие метрики |
| Лог оптимизаций | `logs/evaluations/optimization_log.json` | Персистентный лог фаз |
| Карта оптимизаций | `OPTIMIZATION_ROADMAP.md` | Целевые метрики, план |
| Метрики per-move | `logs/evaluations/metrics.json` | Детальные данные для фильтрации |
| Snapshots | `logs/evaluations/model_rankings_OPT-N.json` | Замороженные метрики фаз |

## Тестовый стенд (3 модели)

| Модель | Тип | Роль |
|--------|-----|------|
| `google/gemini-3-flash-preview` | Fast | Масштаб, скорость |
| `openai/gpt-5-mini` | Premium | Потолок качества |
| `qwen/qwen3-max-thinking` | Deep thinking | Протокол анализа |

## Первичные метрики (Go/No-Go)

| Метрика | Лучше | Жёсткий порог |
|---------|-------|---------------|
| `blunder_rate` | Ниже | +2 п.п. = ОТКАТ |
| `brilliant_rate` | Выше | -3 п.п. (ср.) = ОТКАТ |
| `top3_rate` | Выше | информационная |
| `first_attempt_rate` | Выше | < 0.990 = ОТКАТ |

## Результаты

Лог сохраняется в `logs/evaluations/optimization_log.json`.
Сводная таблица обновляется в `OPTIMIZATION_ROADMAP.md` (секция 8).

## Pipeline оптимизации

Рекомендуемый порядок действий для каждой фазы:

```
1. /evaluate-optimization baseline          ← записать текущее состояние
2. Внести изменение в промпт/код           ← одно изменение!
3. python main.py --headless --bots 3 ...  ← прогнать 45 игр
4. python -m trace_analyzer.metrics --smartbot  ← пересчитать метрики
5. /evaluate-optimization OPT-N "название"  ← оценить эффект
6. Решение: Go / No-Go / Откат
```
