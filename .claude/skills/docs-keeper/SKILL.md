# Docs Keeper Skill

Этот скилл загружает контекст для актуализации двух главных документов проекта.

## Ключевые файлы документации

| Файл | Назначение | Когда обновлять |
|------|-----------|-----------------|
| **`CHANGELOG.md`** | Хронологическая история изменений по категориям Added/Changed/Fixed/Removed | После каждого значимого изменения кода или структуры |
| **`оглавление_llmbot.md`** | Полная карта проекта: модули, агенты, скиллы, команды, зависимости | При добавлении/удалении файлов, модулей, команд CLI |

## Когда ОБЯЗАТЕЛЬНО вызывать

- Добавлен новый `.py` модуль в корень проекта
- Добавлен/изменён агент в `.claude/agents/`
- Добавлен/изменён скилл в `.claude/skills/`
- Добавлен/изменён хук в `.claude/hooks/`
- Добавлены новые аргументы CLI в `main.py`
- Исправлен значимый баг в `bot_runner.py`, `move_parser.py`, `notation_converter.py` и т.д.
- Добавлена новая зависимость (внешняя библиотека)
- Изменена структура директорий проекта
- Добавлены новые тесты (обновить статистику в оглавлении)

## Когда НЕ нужно вызывать

- Правки промптов (`prompts/*.txt`) — не меняют структуру
- Изменения в `logs/` — рабочие данные, не документация
- Минорные рефакторинги без изменения публичного API
- Обновления `.env` или `llm_bot_gui_settings_v2.json`

## Порядок действий при вызове

1. **Загрузи агента** — агент `docs-keeper` (`/update-docs`) выполняет всю работу:

   ```
   /update-docs
   ```

2. **Или вызови явно** с контекстом что изменилось:

   ```
   /update-docs добавлен новый модуль cache.py и CLI-аргумент --cache-ttl
   ```

## Что проверяет агент

```
git status + git diff           →  что изменилось в репозитории
ls *.py, .claude/**, tests/     →  реальная структура проекта
CHANGELOG.md [Unreleased]       →  уже зафиксированные изменения
оглавление_llmbot.md            →  что устарело в карте проекта
```

## Формат записи в CHANGELOG

```markdown
## [Unreleased] — YYYY-MM-DD HH:MM

### Added
- `module.py` — описание зачем добавлен

### Changed
- `bot_runner.py` — что изменилось и зачем

### Fixed
- `move_parser.py` — что было сломано, как исправлено
```

## Правило приоритета документов

Если нет доступа к `git diff` (нет сети, нет git) — используй как источник правды:
1. **`CHANGELOG.md`** — для понимания что уже было до текущей сессии
2. **`оглавление_llmbot.md`** — для понимания актуальной структуры и назначения файлов

Оба файла всегда должны быть консистентны между собой.
