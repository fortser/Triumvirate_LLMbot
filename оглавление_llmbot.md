# 📋 Оглавление проекта Triumvirate LLM Bot v2.2

## 🎯 Описание проекта

**Triumvirate LLM Bot v2.2** — автономный, полностью графический клиент-бот для игры в трёхсторонние шахматы на платформе [triumvirate4llm.com](https://triumvirate4llm.com).

Бот выступает мостом между шахматным сервером и любым провайдером больших языковых моделей (LLM). На каждом ходу бот:
1. Получает состояние доски от сервера
2. Формирует многослойный, настраиваемый промпт
3. Отправляет запрос в выбранную LLM
4. Парсит ответ и извлекает валидный шахматный ход
5. Отправляет ход на сервер

После нажатия кнопки запуска бот играет полностью автоматически до конца партии.

## ✨ Ключевые возможности

### 🤖 Поддержка множества LLM-провайдеров
- **Ollama** — локальные модели (http://localhost:11434/v1)
- **OpenAI API** — GPT-4o, GPT-4o-mini и другие модели
- **Anthropic** — Claude (нативный протокол, не OpenAI-совместимый)
- **OpenRouter** — агрегатор 200+ моделей с отслеживанием стоимости
- **LM Studio** — локальные модели (http://localhost:1234/v1)
- **Кастомный URL** — любой OpenAI-совместимый эндпоинт

API-ключи загружаются из переменных окружения: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, `OPENROUTER_API_KEY`.

### 📊 Форматы ответа LLM
- **Простой текст** — модель отвечает строкой вида `E2 E4`
- **JSON** — модель отвечает объектом `{"move_from": "E2", "move_to": "E4"}`
- **JSON с рассуждением** — модель включает анализ: `{"thinking": "анализ...", "move_from": "E2", "move_to": "E4"}`

### 🛡️ Система надёжности
- **Ретраи с эскалацией** — при нелегальном ходе повторяет запрос (до N попыток). На последней попытке упрощает запрос с прямым списком легальных ходов.
- **Фоллбэк на случайный ход** — если LLM не может принять решение, выбирает случайный легальный ход (можно отключить).
- **Двойной парсинг** — ответ парсится как JSON, затем как текст через регулярные выражения.

### 💰 Отслеживание стоимости
Для OpenRouter автоматически загружаются тарифы моделей. На каждый ход рассчитывается стоимость по input/output/reasoning токенам. Накопительная статистика выводится при остановке.

### 📝 Трассировка ходов
Каждый ход сохраняется как JSON-файл в директорию `logs/game_<id>/move_NNN.json`. Содержит:
- Сырое состояние доски от сервера
- Отрендеренный многослойный промпт
- Все запросы и ответы LLM с количеством токенов и стоимостью
- Все попытки парсинга с промежуточными результатами
- Выбранный ход и ответ сервера

### 🎮 Графический интерфейс
- **NiceGUI** — кроссплатформенный веб-интерфейс на Python
- **Левая панель** — настройки (провайдер, модель, промпты, параметры)
- **Правая панель** — три вкладки:
  - **Игра** — состояние доски, легальные ходы, события
  - **Лог** — полный лог с копированием и очисткой
  - **Лобби** — список активных игр на сервере
- **Диагностика** — кнопки "Ping сервер" и "Проверить LLM"

### 🎨 Персистентные настройки
Все параметры сохраняются в `llm_bot_gui_settings_v2.json` и восстанавливаются при следующем запуске.

## 📄 Описание ключевых файлов

### **main.py** — Точка входа
Инициализирует argparse для обработки аргументов командной строки:
- `--web` — запуск как веб-сервер вместо десктопного окна
- `--host` — хост для веб-сервера (default: 0.0.0.0)
- `--port` — порт для веб-сервера (default: 8090)
- `--settings` — путь к отдельному файлу конфигурации

Запускает GUI через NiceGUI с поддержкой нескольких экземпляров на разных портах и конфигах.

### **constants.py** — Лист-модуль с константами
Определяет:
- **VERSION** — версия приложения (v2.2.0)
- **PROVIDERS** — словарь конфигов всех LLM-провайдеров (base_url, api_key, model, compat, response_format)
- **PROVIDER_ENV_KEY** — маппинг провайдеров на переменные окружения
- **make_bot_name()** — функция для автоматического именования бота из провайдера и модели

Не зависит ни от чего. От него зависят settings, bot_runner, gui.

### **settings.py** — Персистентные настройки и промпты
Обеспечивает:
- Чтение/запись JSON-конфига (`llm_bot_gui_settings_v2.json`)
- Загрузку дефолтных промптов из `prompts/*.txt` с fallback на встроенные строки
- Класс **Settings** с полями: провайдер, модель, API-ключ, параметры LLM, флаги поведения
- Функция **get_response_format()** для определения формата ответа

### **llm_client.py** — Asyncio HTTP-клиент для LLM
Класс **LLMClient** с методом **ask()**:
- Поддерживает OpenAI-совместимый протокол (GPT-4, Ollama, LM Studio, OpenRouter)
- Поддерживает нативный протокол Anthropic с использованием `messages` и `thinking` blocks
- Возвращает кортеж (response_text, full_response_body) для доступа к usage и cost

### **arena_client.py** — REST API клиент арены
Класс **ArenaClient** для взаимодействия с сервером Triumvirate:
- **join()** — присоединиться к игре, получить токен и цвет
- **get_state()** — получить текущее состояние доски, легальные ходы, инфо о шахе
- **move()** — отправить ход
- **resign()** — сдаться
- **health()** — проверить доступность сервера

### **pricing.py** — Менеджер тарифов OpenRouter
Класс **PricingManager**:
- **fetch_openrouter()** — загружает цены для моделей из OpenRouter /api/v1/models
- **calculate_cost()** — рассчитывает стоимость для prompt/completion/reasoning токенов
- Для других провайдеров возвращает нули

### **prompt_builder.py** — Сборщик промптов
Класс **PromptBuilder** с методом **build()**:
- Собирает список сообщений из системного промпта и пользовательского шаблона
- Заполняет плейсхолдеры: `{{move_number}}`, `{{current_player}}`, `{{legal_moves}}`, `{{board}}`, `{{check}}`, `{{position_3pf}}`, `{{last_move}}`
- Поддерживает как серверную нотацию (A1–L12), так и TRIUMVIRATE v4.0

### **move_parser.py** — Парсер ответов LLM
Класс **MoveParser** с методом **parse()**:
- **JSON-режим** (json / json_thinking) — ищет ключи `move_from` и `move_to` в JSON-объекте
- **Текстовый режим** (simple) — regex-поиск координат в тексте ответа
- Обрабатывает обе нотации: серверная (A1) и TRIUMVIRATE (W3/B3.0)
- Поддерживает промоушен пешек (q, r, b, n)
- Санитизирует некорректный JSON от моделей (неэкранированные управляющие символы)

### **tracer.py** — Трассировщик ходов
Класс **MoveTracer**:
- Собирает полный pipeline trace для одного хода
- Сохраняет JSON-файлы в `logs/game_<id>/move_NNN.json`
- Содержит: сырой стейт, промпты, все LLM-запросы/ответы, попытки парсинга, выбранный ход, ответ сервера
- Отслеживает стоимость, время, количество ретраев

### **bot_runner.py** — Основной игровой цикл
Класс **BotRunner** с методом **run()**:
- Оркестрирует весь игровой процесс: подключение к арене, поллинг, LLM-запросы, парсинг, отправку ходов
- Реализует **ретраи с эскалацией** при нелегальных ходах
- Фоллбэк на **случайный ход** при исчерпании попыток
- Интегрирует: llm_client, arena_client, prompt_builder, move_parser, tracer, pricing
- Использует asyncio для неблокирующих операций

### **gui.py** — NiceGUI графический интерфейс
Функция **create_gui()** строит весь интерфейс:
- **Левая панель** — форм-элементы для настроек (провайдер, модель, API-ключ, промпты, флаги)
- **Правая панель** — три вкладки (Игра, Лог, Лобби)
- **Кнопки** — Start, Stop, Ping сервер, Проверить LLM
- **Callbacks** — синхронизация между GUI и bot_runner

### **notation_converter.py** — Конвертер нотаций
Предвычисленные таблицы для O(1) конверсии:
- **Серверная нотация** — A–L (колонки) + 1–12 (строки), например A1, E4, L12
- **TRIUMVIRATE v4.0** — радиально-кольцевая нотация, например W3/B3.0, C/W.B
- Функции: **convert_board()**, **convert_legal_moves()**, **convert_move_back()**, **to_triumvirate()**

## 🔍 trace_analyzer — Веб-инструмент для анализа логов

### Назначение
Визуальный анализ и отладка trace-логов бота. Позволяет быстро находить ошибки, сравнивать модели, изучать reasoning и экспортировать данные.

### Функционал (4 вкладки)
1. **Overview** — дашборд с метриками по играм, scatter-графики, аномалии
2. **Moves Table** — сортируемая таблица всех ходов с фильтрами и цветовыми индикаторами
3. **Thinking Gallery** — карточки reasoning с мультивыбором и экспортом в MD/JSON
4. **Move Detail** — детальный разбор хода с 7 раскрываемыми секциями

### Структура
```
trace_analyzer/
├── app.py                 # Точка входа, layout
├── data_loader.py         # Загрузка trace-файлов
├── export_utils.py        # Экспорт в Markdown / JSON
├── requirements.txt       # nicegui>=2.0
└── views/
    ├── overview.py        # Дашборд
    ├── moves_table.py     # Таблица
    ├── thinking_gallery.py # Галерея reasoning
    └── move_detail.py     # Детали хода
```

## 🚀 Команды запуска

```bash
# Десктопное окно (1440×900)
python main.py

# Веб-сервер на порту 8090
python main.py --web

# Веб-сервер на другом порту
python main.py --web --port 9000

# Несколько ботов одновременно с разными конфигами
python main.py --web --port 8090 --settings bot_gpt4.json
python main.py --web --port 8091 --settings bot_claude.json

# Запуск на headless-сервере (Ubuntu без GUI)
python main.py --web --host 0.0.0.0 --port 8090

# Запуск trace analyzer
cd trace_analyzer
python app.py --logs ../logs --port 8091
```
