# Triumvirate LLM Bot — GUI v2.2

## Суть и предназначение проекта
Данный проект представляет собой интеллектуального бота, работающего на базе больших языковых моделей (LLM), предназначенного для игры в шахматы для троих (Three-Player Chess) на платформе арены [triumvirate4llm.com](https://triumvirate4llm.com). 

Бот подключается к игровому серверу через REST API, получает текущее состояние партии, формирует детализированный промпт (запрос) с описанием ситуации на доске и отправляет его в выбранную LLM. Получив ответ от нейросети, бот парсит выбранный ею ход и передаёт его на сервер, организуя таким образом полностью автономный игровой процесс. Управление настройками и аналитика осуществляются через встроенный графический веб-интерфейс на базе фреймворка [NiceGUI](https://nicegui.io/).

---

## Возможности и функционал
- **Кросс-модельная поддержка LLM**: Пресеты для работы с OpenAI, Anthropic (нативный API), OpenRouter, локальными движками (Ollama, LM Studio) и кастомными OpenAI-совместимыми эндпоинтами.
- **Интеллектуальная обработка ходов**: Встроенный парсинг ответов LLM в форматах `json`, `json_thinking` и обычном текстовом (`simple`). Поддерживается извлечение полей `move_from`, `move_to` и `promotion` с обходом частых ошибок нейросетей (например, неэкранированных символов в JSON).
- **Поддержка двух нотаций**: 
  - Классическая серверная нотация (буквы A-L, цифры 1-12, например `A1`, `L12`).
  - Радиально-кольцевая нотация TRIUMVIRATE v4.0 (например, `W3/B3.0`, `C/W.B`), улучшающая пространственное восприятие позиции нейросетями.
- **Автоматические ретраи (повторные попытки)**: В случае некорректного, галлюцинированного или незаконного хода (illegal move) бот автоматически делает повторный запрос к LLM, повышая `temperature` (креативность) для избежания зацикливаний.
- **Фоллбэк стратегия**: В случае многократных неудачных ответов LLM, бот может сделать случайный легальный ход (`fallback_random`), чтобы избежать технического поражения по тайм-ауту.
- **Прозрачность и контроль затрат**: Парсинг и логирование использования токенов (prompt, completion, reasoning) и автоматический расчёт стоимости через API OpenRouter.
- **Мощная система логов (Трейсинг)**: Модуль трейсинга сохраняет детальные JSON-отчёты (трейсы) о каждом сделанном ходе в папке `logs/`, включая сырые ответы сервера, полные промпты и все ответы от LLM.
- **Удобный GUI (Graphical User Interface)**: Настройка промптов, ключей API, правил и параметров генерации прямо в удобном десктопном или веб-интерфейсе.

## Полное описание логических модулей

Взаимосвязанные модули выстроены в четкую архитектуру, разделяющую игровую логику, сетевое взаимодействие и преобразование данных.

1. **`main.py`**  
   *Точка входа в приложение.* Инициализирует настройки, парсит аргументы командной строки и запускает графический интерфейс пользователя (NiceGUI) в десктопном или веб-режиме (встроенный сервер).

2. **`bot_runner.py`**  
   *Основной игровой asyncio-цикл (оркестратор).* Управляет всем процессом: поллинг (опрос) состояния сервера, ожидание своего хода, обращение к генератору промптов и LLM-клиенту, парсинг хода, обработка ошибок, выполнение ретраев и отправка итогового хода обратно на сервер.

3. **`arena_client.py`**  
   *HTTP-клиент для взаимодействия с API арены Triumvirate.* Содержит методы интеграции с сервером по REST: аутентификация (`join`), получение стейта игры (`state`), совершение хода (`move`), а также пропуск ожидания и получение списка игр.

4. **`llm_client.py`**  
   *HTTP-клиент для связи с языковыми моделями.* Реализует поддержку двух ключевых протоколов общения с LLM: стандарта OpenAI-compatible (через эндпоинт `/chat/completions`) и нативного формата Anthropic (`/v1/messages`).

5. **`prompt_builder.py`**  
   *Сборщик многослойных промптов.* Берёт сырое состояние (board, legal moves, previous move, check events), форматирует в человекочитаемый текст или кастомную нотацию, применяет шаблоны системного и пользовательского промпта и склеивает это в единый массив сообщений для LLM (`messages`).

6. **`move_parser.py`**  
   *Парсер ответов LLM.* Регулярными выражениями или через `json.loads` извлекает координаты (откуда, куда) и информацию о превращении фигуры. Имеет встроенный "санитизатор" JSON, исправляющий битые переносы строк и маркдаун-обёртки ` ```json ... ``` `.

7. **`notation_converter.py`**  
   *Драйвер системы координат.* Предвычисляет и конвертирует координаты сервера (A-L, 1-12) в радиальную нотацию Triumvirate v4.0 (сектора W, B, R) и обратно за время O(1).

8. **`pricing.py`**  
   *Менеджер финансов.* Обращается к OpenRouter API (`/models`), скачивает стоимости токенов и калькулирует финансовые затраты (USD) на каждый конкретный ход бота.

9. **`tracer.py`**  
   *Слой Observability.* Собирает всю информацию за один цикл хода (от состояния сервера до количества токенов), агрегирует её и сбрасывает в локальный JSON-файл в `logs/game_<id>__<model>/move_NNN.json`. Незаменим для дебага LLM.

10. **`settings.py`**  
   *Менеджер конфигураций.* Загружает и сохраняет пользовательские настройки и пути к шаблонам ответов в персистентный файл `.json`, восстанавливая дефолтные промпты при первой загрузке.

11. **`constants.py`**  
   *Хранилище констант.* Содержит версию приложения, дефолтные пресеты параметров для разных LLM-провайдеров и алгоритм автоматической генерации имени бота.

12. **`gui.py`**  
   *(Самый массивный модуль)* Отрисовка элементов UI инструментами `nicegui`. Создает формы для редактирования API-ключей, ползунки температуры LLM, текстовые поля для редактирования промптов файлов в `/prompts`, а также выводит интерактивный лог событий в реальном времени.

## Входные и выходные форматы

### Входные данные (Inputs)
- **Конфигурация:** Файл `llm_bot_gui_settings_v2.json`, текстовые файлы с шаблонами (например, `prompts/system_prompt.txt`).
- **Стэйт сервера (REST JSON):** Данные от Triumvirate API (словарь, содержащий состояние `board` (координата фигуры, цвет, тип), список дозволенных ходов `legal_moves`, текущего игрока, состояние шаха и т.д.).
- **Ответы LLM:** Текст или JSON-текст от нейросетей, содержащий анализ (поток мыслей) и финальный ход.

### Выходные данные (Outputs)
- **Ход на Сервер (REST JSON):** Запрос на `/api/v1/move`, содержащий данные хода:
  ```json
  {
      "from": "A2", 
      "to": "A3", 
      "promotion": "queen", 
      "move_number": 42
  }
  ```
- **Трейс-логи (Local JSON):** Файлы `logs/.../*.json`, хранящие полную телеметрию о ходе (используемую промпт-цепочку, сырые ответы от API, время ответа, потраченные USD‑центы, итог).

## Запуск проекта и CLI параметры

Запускать проект необходимо через главный скрипт `main.py`.

### Параметры командной строки (CLI)
Скрипт `main.py` поддерживает аргументы через модуль `argparse`:

| Аргумент | Описание | Присоединенное значение / Тип |
| :--- | :--- | :--- |
| `--web` | Запустить GUI в виде веб-сервера (в браузере), а не разворачивать нативное десктопное окно. | Флаг (Store True) |
| `--host` | Установить хост для веб-сервера (имеет смысл только в режиме `--web`). | Строка, по умолчанию: `0.0.0.0` |
| `--port` | Установить порт для веб-сервера (имеет смысл только в режиме `--web`). | Число (int), по умолчанию: `8090` |
| `--settings`| Указать путь к кастомному JSON-файлу настроек. | Строка (путь к файлу). Позволяет запускать *несколько ботов одновременно* из одной директории, передавая каждому свой конфиг. |
| `-h`, `--help`| Вывести справочную информацию по аргументам скрипта. | Встроенный функционал argparse. |

### Примеры команд для запуска
1. **Запуск стандартного оконного Desktop-приложения:**
   ```bash
   python main.py
   ```
2. **Запуск в качестве демона веб-сервера (доступ через http://localhost:8090):**
   ```bash
   python main.py --web
   ```
3. **Запуск веб-сервера на определённом порту:**
   ```bash
   python main.py --web --port 9000
   ```
4. **Запуск мульти-обработчиков (двух независимых ботов с разными настройками):**
   ```bash
   # Терминал 1
   python main.py --web --port 8091 --settings bot_white_config.json
   
   # Терминал 2
   python main.py --web --port 8092 --settings bot_black_config.json
   ```
